# E1 · Первая польза за 30 секунд

**Owner:** product + frontend + backend · **Волна:** СЕЙЧАС

## Зачем

Дыра №1 (overview §10 «низкое удержание без быстрых побед»). Сейчас: регистрация → «добавь 5 каналов»
(каких?) → дни тишины → ушёл. Надо: регистрация → выбрал тему → готовый набор каналов в 1 клик →
сразу видит «вот что завирусилось за 24ч».

## Решения, зафиксированные при планировании

- Пак ≠ 50 строк watchlist в лимите юзера: watchlist-строки пака помечаются `pack_slug` и **не считаются**
  в `Resource.CHANNELS`; лимитируется число паков (Free=1, Pro=5 — константы).
- Instant value питается от **системного showcase-тенанта** (подписан на все паки) — не ждём, пока
  pipeline нового юзера прогреется. Тот же тенант потом питает витрину-канал (E3) — двойная отдача.
- Free-rework по ценам (Free = только паки) — это E5/TASK-049; здесь только механика задержки.

## Финальная точка эпика

Новый человек за минуту после регистрации видит релевантные вирусные сигналы по своей теме,
ничего вручную не настраивая. Метрика «регистрация → первый увиденный сигнал» (E6) подтверждает < 60 сек.
