# E0 · Выживание и ops-фундамент

**Owner:** infra + backend · **Волна:** СЕЙЧАС · **Источник:** [pain-points P1/P2/P3/P4/P7/P8](../../architecture/pain-points.md)

## Зачем

Один инцидент с данными обнуляет всё остальное; молчащий пул аккаунтов = продукт молчит под клиентами;
непомеренная задержка = непомеренное главное обещание продукта. Это дешёвая профилактика, которая
делается **до** включения денег.

## Финальная точка эпика

Бэкап восстановлен хотя бы один раз руками по runbook; пул ≥3 аккаунтов с мониторингом;
p50/p95 задержки на приборе; эмбеддинги не считаются повторно. После этого продукт можно
не стыдно показывать платящим.
