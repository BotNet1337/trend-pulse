# E4 · Деньги без трения (автопродление)

**Owner:** backend + product · **Волна:** после E1–E3

## Зачем

У крипты нет автосписания — каждое ручное продление (learnings task-010) = точка оттока. Task-027 шлёт
напоминания, но платёж ручной. Убираем точки трения: реже платить (год/квартал), легче платить
(one-click), мягче истекать (grace).

## Задачи (defined)

| Task | Что | DoD кратко |
|------|-----|------------|
| TASK-047 · Годовые/квартальные планы | `BillingPeriod` QUARTER/YEAR со скидкой 20–30% (константы в `billing/plans.py`), invoice-создание + IPN уже period-aware (проверить), UI выбора периода | можно купить год со скидкой; деньги вперёд |
| TASK-048 · Renewal one-click + grace + partially_paid | Напоминание (task-027) содержит прямую ссылку «продлить» → создаёт invoice на текущий план/период; grace-период 3 дня мягкой деградации вместо обрыва в Free (`effective_plan` учитывает grace-окно, константа); обработка NOWPayments `partially_paid` → письмо «доплати X» со ссылкой | продление в один клик из письма/алерта; истечение не обрывает резко; недоплата не теряется молча |

## Финальная точка эпика

Есть годовой план со скидкой, напоминания ведут на оплату в один клик, grace вместо обрыва,
`partially_paid` обрабатывается. Метрика (E6): доля продлений без ручного захода в дашборд растёт,
churn на дате истечения падает.
