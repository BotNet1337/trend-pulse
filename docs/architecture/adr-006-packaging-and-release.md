# ADR-006 — Packaging & release (OCI artifacts via ORAS → `release` repo)

- Status: **Proposed (forward-looking — не реализуем сейчас, учитываем при проектировании)**
- Date: 2026-06-08
- Context: [build-and-release.md](./build-and-release.md), [high-level-architecture.md](./high-level-architecture.md)

## Context

`botnet/` — монорепо с несколькими приложениями/ботами (`apps/trendPulse` — первый). В будущем мы хотим простой путь деплоя на VPS: каждый бот публикуется как OCI-артефакт через **ORAS** (OCI Registry As Storage) в репозиторий `botnet/release`, который агрегирует все боты как зависимости и собирает **один удобный VPS-деплой**. Сейчас это НЕ реализуем — но проектируем так, чтобы переход был бесшовным.

## Decision (направление)

1. **Каждое приложение — самодостаточный, версионированный OCI-образ.** Команда запуска — снаружи (compose), не зашита в образ. Один образ покрывает роли (`api`/`worker`/`beat`).
2. **Дистрибуция через ORAS.** Образы + сопутствующие манифесты (compose-фрагмент сервиса, provisioning, метаданные) пушатся как OCI-артефакты в registry; `botnet/release` подтягивает их как **pinned-зависимости** и собирает единый bundle для VPS.
3. **`release`-репо — точка агрегации.** Он не содержит код ботов, только декларацию: какие артефакты каких версий собрать в один деплой (сети, provisioning-ордер, env-контракт).
4. **Конфигурация — строго 12-factor / env.** Никаких build-time привязок к хосту, путям, секретам. Секреты — из Ansible/vault на целевой машине (ADR-005). Это делает образ переносимым между dev/VPS/release-bundle без пересборки.

## Что фиксируем для текущих задач (чтобы не закладывать долг)

- **task-001 / task-012:** не хардкодить host-пути/порты в образ; конфиг только через `deploy.env`/`sensitive.env`; per-service compose-фрагменты должны быть переносимы в release-bundle; provisioning — отдельные образы/шаги.
- **Версионирование:** образы тегируются semver; деплой ссылается на pinned-версии.
- **Сетевой контракт:** имена сетей/сервисов стабильны (network-design), чтобы release-bundle переиспользовал их.

## Consequences

- (+) Будущий мульти-бот VPS-деплой собирается из готовых OCI-артефактов без пересборки исходников.
- (+) Чёткое разделение: код в `apps/*`, дистрибуция/агрегация в `release/`.
- (+) Воспроизводимость через pinned semver-артефакты.
- (−) Дисциплина 12-factor/версионирования нужна с самого начала (иначе host-coupling всплывёт при миграции на ORAS).
- (−) Доп. инфраструктура (registry, ORAS-пайплайны) — позже, не сейчас.
- Влияет (как ограничение дизайна) на: **task-001** (образ/compose переносимость), **task-012** (IaC/деплой), будущие боты в `botnet/apps/*`.
